Our Methodology
We are practitioners of Domain-Driven Design (DDD), Behavior-Driven Development (BDD), and Continuous Delivery (CD)!
Domain-Driven Design


DDD is an approach to software development that centers the development on programming a domain model that has a rich understanding of the processes and rules of a domain.

Behavior-Driven Development


BDD is a test-first approach that tests the actual behavior of the system from the end users perspective.

Continuous Delivery


CD is a software development discipline where you build software in such a way that the software can be released to production at any time.
